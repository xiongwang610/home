Bean配置项

  • id:唯一标识
  • class:实现类(必须配置)
  • scope:范围作用域
  • constructor or arguments:构造器注入
  • properties:参数注入
  • Autowring mode:自动装配模式
  • lazy-initialization:懒加载模式
  • Initialization/destruction:初始化方法或销毁方法

AOP

AOP一些概念

  • Aspect：一个关注点的模块化，这个关注点可能横切多个对象
  • JointPoint: 程序执行过程中的某个特定点
  • Advice:在切面的某个特定的连接点上执行动作
  • Pointcut:匹配连接点的断言，在AOP中通知和一个切入点的表达式关联
  • Introduction:在不修改类代码的前提下，为类添加新的方法和属性
  • TargetObject:被一个或者多个切面通知的对象
  • AOPProxy:AOP框架创建对象，用来实现切面契约
  • Weaving:把切面连接到其他的应用程序类型或者对象上，并创建被通知对象，编译时织入，类加载时织入，执行时织入

Advice

  1. 前置通知(Before Advice)：joint point之前执行但不能阻止连接点前执行
  2. 返回后通知(After returning Advice):joint point 正常完成后执行的通知
  3. 抛出异常后通知（After throwing Advice):在方法抛出异常退出执行的通知
  4. 后通知(After Advice): 某连接点退出时执行(不论是否正常退出)
  5. 环绕通知(Around Advice):包围一个连接点前后

Spring AOP

aspect pointcut advice配置

<bean id="aspect" class="通知类" />
<bean id="target" class="目标类" />
<aop:config>
    <aop:aspect id="Aop" ref="Aspect">
        <aop:pointcut expression="excution(*包名).*function(..)" id="pointcut"/>
        <aop:before method="指定方法" pointcut-ref="pointcut"/>
        <aop:after-returning method="指定方法" pointcut-ref="pointcut"/>
        <aop:after-throwing method="指定方法" pointcut-ref="pointcut"/>
        <aop:after method="指定方法" pointcut-ref="pointcut"/>
        <aop:around method="指定方法" pointcut-ref="pointcut"/>
    </aop:aspect>
</aop:config>
//使用around时方法第一个参数必须是ProceedingJoint类型
public Object function(ProceedingJointPoint pjp)    throws Throwable{
    //before....
    Object result = pjp.proceed();
    //after...
    return result;
}

注解实现AOP

//首先在配置文件中配置
//<context:component-scan base-package="扫描该包下的注解"
/**
 * 切入点
 * execution(* *(..))匹配
 * @annotation(注解类)
 * 
 */
@Pointcut("@annotation(whut.glxiang.cbim.annotation.Operate)")
public void operate(){

}
/**
 * 环绕通知
 * @param point
 * @return
 */
@Around("operate()")
public Object around(ProceedingJoinPoint point){
    Object object = null;
    //前置
    object = point.proceed();
    //后置   
    return object;
}
